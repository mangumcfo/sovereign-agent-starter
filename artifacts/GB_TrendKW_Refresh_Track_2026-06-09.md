# Trend-KW Refresh — Metalayer Track (2026-06-09)

> **What this is:** the rails for keeping chapter-level keywords aligned to what's trending on x.com / social / Amazon-KU search, **volume by volume**, so the catalog stays relevant. GB lays the track + passes KM each prompt; KM relays to G on x.com; GB folds the return; it surfaces at final manuscript review (human gate) as we've always done. ∞Δ∞

## The loop (per volume)
1. **GB** builds the per-volume prompt, populated with that book's REAL enriched chapters (title + promise + current chapter-KW from the index).
2. **KM** relays it to **G on x.com**.
3. **G** returns, per chapter: trend-aligned keywords + a one-line trend note (rising/fading) + any emerging term to add.
4. **GB** folds the return into the extraction index — chapter-level `keywords`, marked `generated`, **dated**, `kw_source: G-trend-relay <date>`; keeps a diff vs prior KW.
5. **Human review (KM)** sees the updated KW + G's trend note + provenance at **final manuscript review** — ratify / adjust. (Data requirement: every KW change is dated, sourced, and review-visible — no silent swaps.)

## Data requirements for human review (as always)
- Each chapter card shows: **current KW**, **proposed KW (G-trend)**, **trend note**, **date + source**.
- Nothing overwrites silently; prior KW retained in the diff until KM ratifies.
- Manuscript titles/promises are NOT changed by this loop — KW only.
