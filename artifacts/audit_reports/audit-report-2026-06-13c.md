# Sovereign Stack Audit — 2026-06-13c (W6 confirming sweep, post-W5)

*7 dimensions audited (security, performance, code quality, test coverage, dependencies, architecture, constitutional conformance), with every finding adversarially verified — false positives refuted and removed.*

## Executive Summary

**Overall health: 78/100.** This is a mature, defensively-engineered stack: the hash-chained obligation ledger has a correct cross-process write-fence, the auth/crypto core is hardened (constant-time compare, CORS allowlist, CSRF stop, fail-closed gates), and a recent (2026-06-10 → 2026-06-13) audit sweep already remediated most prior security and performance holes. 195 tests pass in ~7.7s with zero flaky/failing. The score is held below the 90s by **zero CRITICAL but five HIGH** findings clustered around one recurring theme: **propagation debt — a defense done right in one place and silently omitted on a sibling path.** That theme appears across every dimension: owner-gating applied to four disposition routes but missed on two breath-gate routes; `get_ledger_root()` adopted by the API/executor but bypassed by six writer scripts; the resilient YAML loader adopted by most readers but not `extrusion_validate`; the principal-binding migration completed everywhere except `atrium_apply.py`, where a swallowed unauthenticated close marks a proposal "applied" while the obligation stays open. The single most important issue is that last one (`atrium_apply.py`): a silent seal-chain-vs-ledger partial-state divergence reported as success, the constitutional §3/§4 sibling of the bugs the team has been hunting.

**3. Performance — ledger `_entries()` re-parses the entire NDJSON on every write** · `src/sovereign_agent/obligations/ledger.py:213-231, 226` · `_entries()` reads the whole NDJSON (`read_text().splitlines()` + `json.loads` per line) on each cache miss. O(n) per mutation, unbounded. · **Fix:** append-aware tail-parse (seek to prior offset on a size-only grow, parsing only the appended tail); longer term, NDJSON segment rotation + compacted state snapshot. · half a day (tail-parse); 1-2 days (rotation/snapshot) · *Verifier: CONFIRMED against code and on-disk data; the executor shares the same root/flock, confirming the frequent-write premise.*
