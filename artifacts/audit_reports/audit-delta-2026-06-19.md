## New findings

### MED
- **[MED][constitutional: silent failure / code quality] scripts/review_ready_contract.py:357 (`_check_substance`) and :318 (`_check_artifact_package`)** — the `/seeit` coverage check is **hardcoded to S3V3**, not scoped to `book_id`. In `_check_substance`, `seeit_n = len(re.findall(r'"id":"s3v3-', ...))`; in `_check_artifact_package`, `seeit_ok = any(tok in txt for tok in ("helix", "s3v3", "vol_03_helix"))`. The `book_id` parameter is accepted but never used to scope the seeit lookup. Why it matters: for **any volume other than S3V3**, `seeit_n`/`seeit_ok` is structurally 0/False, so both gates emit a `/seeit` issue and the contract goes NOT-READY for reasons unrelated to that book's actual seeit pages — and conversely, an S3V3-named exercises.py would satisfy the seeit floor for a *different* book sharing the same `exercises.py`. This is a false-RED (and a cross-book false-attribution) baked into a blocking gate; it will mis-gate the very next non-Helix volume. It is conservative w.r.t. the constitution (it blocks rather than false-readies, so not CRITICAL/HIGH), but it is a real correctness defect in the gate logic. Cross-verified against live code: yes (`book_id` unused in both functions). **FIX:** derive the seeit token from `book_id` (e.g. map `book_id` → its seeit slug) and match `f'"id":"{slug}-'` / that slug in `exercises.py`, so the seeit floor measures *this* book's pages; until then, scope the hardcoded S3V3 match behind an explicit `if book_id is the helix volume` guard and return a `skipped` (not RED) seeit sub-result for other books.

## High-risk targets — clean

**review_ready_contract.py gate engine — cannot be silently bypassed:**
- `evaluate()` appends all six new gates into the single `checks` list, and `ready = all(c["pass"] for c in checks)` — a new gate is AND-ed into readiness; there is no path where a new gate is computed but excluded from the verdict. `main()` returns `0 if review_ready else 1` and `__main__` does `raise SystemExit(main())`, so a failing gate exits non-zero loudly.
- **No false-ready on missing/empty inputs.** Every new gate returns `pass=False` when its input is absent: `bdir is None` → RED; no manuscript → RED; no PDF in `final/` → RED; no `toolchain.json` → RED; empty `producers` `{}` → RED (the `for need in (...)` loop adds "missing provenance"); invalid JSON → RED. Empty-manuscript path verified end-to-end: empty file → `toc_format` RED → `production_standards` gate RED.
- **Delegated lints fail loud, not silent.** `_check_render_fidelity` and `_check_production_standards` wrap the `import` + `.lint()` in `except Exception as e: return {... "pass": False, "gap": f"... failed: {e}"}` — a poppler-missing or import error surfaces as RED with the error text, never a swallowed pass.
- `_vkey` numeric version sort is correct (`manuscript_v1.10` ⇒ `(1,10)` > `(1,9)`), so the "latest manuscript" the gates read is genuinely the highest version, closing a string-sort regression rather than introducing one.
- The gates are read-only over the book dir; the only write in `main()` is the `review_ready/{book_id}.json` record and the review-packet mint, both gated behind `review_ready` being true — no ledger owner is taken from a request body (this is a CLI tool, not an API route), so no principal-spoof surface.

**hopper.py collect-all + recency-cap — sound, removes a silent drop:**
- The removed in-loop `break` (old `if len(cards) >= _MAX_CARDS*2: break`) was itself the silent-drop bug: it truncated the feed in *file order* (oldest-first), discarding the newest high-priority cards before they could sort up. The fix collects ALL cards, then sorts recency-desc then priority (two stable sorts), then caps `cards[:_MAX_CARDS*2]` AFTER the Sent-to-Packet filter — so the cap keeps the newest live high-priority cards. Converts a silent newest-card drop into a deterministic keep-newest.
- Cards missing `ts` default to `""` and sort last within their priority bucket, but the priority sort runs second and is stable, so a high-priority card without `ts` still beats every normal card; with the `*2` cap headroom it is not dropped. No high-priority input is silently lost.
- No principal/ownership surface in this read path; no gate model touched.
